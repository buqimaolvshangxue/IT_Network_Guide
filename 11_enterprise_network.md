## 11.2 企业网络 vs 家庭网络

| 特性 | 家庭网络 | 企业网络 |
|---|---|---|
| **设备数量** | 5-20 台 | 几十到几万台 |
| **管理方式** | 你自己管 | IT 部门统一管理 |
| **IP 分配** | 一个路由器 DHCP | 多个 DHCP 服务器/静态分配 |
| **网络隔离** | 通常没有 | VLAN 隔离不同部门 |
| **安全策略** | 基本没有 | 防火墙、准入、审计 |
| **Wi-Fi 认证** | 密码 | 802.1X（用户名+密码/证书） |
| **上网控制** | 没有 | 上网行为管理/代理 |

## 11.3 典型企业网络架构

```
互联网
  │
  ↓
┌──────────────┐
│   防火墙      │  ← 第一道防线，控制进出流量
│  (Fortinet/   │
│   Palo Alto)  │
└──────┬───────┘
       │
┌──────┴───────┐
│   核心交换机   │  ← 网络中枢，连接所有区域
└──┬───┬───┬───┘
   │   │   │
   ↓   ↓   ↓
┌────┐┌────┐┌────┐
│VLAN││VLAN││VLAN│
│ 10 ││ 20 ││ 30 │
│办公 ││研发 ││访客 │
└────┘└────┘└────┘

另外还有：
┌──────────┐  ┌──────────┐  ┌──────────┐
│  DMZ 区   │  │ 服务器区  │  │  无线控制器 │
│(对外服务) │  │(内部系统) │  │ (管理AP)  │
└──────────┘  └──────────┘  └──────────┘
```

### 关键区域

| 区域 | 说明 |
|---|---|
| **DMZ（隔离区）** | 放对外服务（官网、邮件服务器），能被外网访问但和内网隔离 |
| **办公区** | 员工日常办公的网络 |
| **研发区** | 开发人员网络，通常权限更大 |
| **服务器区** | 内部系统（OA、ERP、Git 等） |
| **访客区** | 给来访客人用的 Wi-Fi，和内网完全隔离 |

## 11.4 VLAN —— 网络隔离

### 为什么要 VLAN？

如果所有人都在同一个网络里：
- 研发能访问财务的电脑
- 访客能访问内部服务器
- 一台电脑中毒，整个网络都受影响

🔑 **VLAN 就是在一个物理网络上划分出多个"虚拟局域网"，互相隔离。**

### VLAN 的效果

```
同一个交换机，但：
  VLAN 10（办公）：192.168.10.0/24  ← 只能访问办公资源
  VLAN 20（研发）：192.168.20.0/24  ← 能访问研发服务器
  VLAN 30（访客）：192.168.30.0/24  ← 只能上网，不能访问内网

VLAN 10 的电脑 ping VLAN 20 的电脑 → ❌ 不通（除非路由器/防火墙允许）
```

### 你需要知道的

- 你的电脑连上公司网络后，会被分配到某个 VLAN
- 不同 VLAN 之间默认不通，需要通过路由器/防火墙才能互访
- 如果你访问不了某个内部系统，可能是 VLAN 权限问题 → 找 IT

## 11.5 企业 Wi-Fi（802.1X）

### 为什么公司 Wi-Fi 要输用户名密码？

家里 Wi-Fi 用的是 **WPA2-PSK**（Pre-Shared Key）——所有人用同一个密码。

公司 Wi-Fi 用的是 **WPA2-Enterprise（802.1X）**——每个人用自己的账号密码登录。

> 🔰 **802.1X 是什么？** 它是一种**网络准入认证标准**（IEEE 制定的），核心思想是：设备连上网络之前，必须先证明"你是谁"。家里 Wi-Fi 只需要一个共享密码，而 802.1X 要求每个人用自己的用户名+密码（或数字证书）登录，就像进公司要刷自己的工牌一样。


| 特性 | WPA2-PSK（家用） | WPA2-Enterprise（企业） |
|---|---|---|
| 认证方式 | 共享密码 | 每人独立账号 |
| 员工离职 | 要改密码通知所有人 | 禁用他的账号就行 |
| 审计 | 不知道谁连了 | 知道每个人的连接记录 |
| 安全性 | 一般 | 高 |

### 连接方式

```
公司 Wi-Fi 连接时：
  SSID：Company-WiFi
  安全性：WPA2 Enterprise
  认证方式：PEAP（账号密码）或 EAP-TLS（客户端证书）
  用户名：你的工号或邮箱
  密码：你的域账号密码

  有时还需要：
  - 安装公司的 CA 证书
  - 通过 MDM（移动设备管理）注册设备
```

## 11.6 企业 VPN

### 什么时候用？

- 在家远程办公，需要访问公司内网
- 出差时连公司系统
- 在外面访问公司的 Git/OA/邮件

### 常见企业 VPN 方案

| 方案 | 说明 |
|---|---|
| **Cisco AnyConnect** | 最常见的企业 VPN 客户端 |
| **GlobalProtect** | Palo Alto 防火墙配套 |
| **FortiClient** | Fortinet 防火墙配套 |
| **OpenVPN** | 开源方案 |
| **WireGuard** | 新兴方案，部分企业开始用 |
| **SSL VPN** | 通过浏览器访问，不需要装客户端 |

### 使用方式

```
通常 IT 部门会给你：
1. VPN 客户端安装包
2. VPN 服务器地址
3. 你的账号密码（通常和域账号一样）
4. 可能还有证书文件

连接后：
- 你的电脑会多一个虚拟网卡
- 获得一个公司内网 IP（如 10.x.x.x）
- 可以访问公司内部系统
```

### 全局 VPN vs 分流 VPN

| 模式 | 说明 | 影响 |
|---|---|---|
| **全局（Full Tunnel）** | 所有流量都走 VPN | 上网也走公司网络，公司能看到所有流量 |
| **分流（Split Tunnel）** | 只有访问公司网段才走 VPN | 上网走自己的网络，更快 |

🔑 **大部分公司用全局模式**（安全考虑）。如果你发现连了 VPN 后上网变慢，就是因为所有流量都绕了一圈公司网络。

### 💻 查看 VPN 状态

```bash
# 查看路由表，看流量走哪
netstat -rn     # Mac
ip route        # Linux

# 查看是否有 VPN 虚拟网卡
ifconfig        # 看有没有 utun0/tun0 之类的接口

# 查看当前公网 IP（判断是不是走了 VPN）
curl ifconfig.me
```

## 11.7 域控（Active Directory）

### 一句话概括

🔑 **域控（AD）就是公司的"中央账号管理系统"——一个账号登录所有系统。**

### 你会遇到的场景

| 场景 | 背后是域控 |
|---|---|
| 开机输入用户名密码 | 域账号认证 |
| 连 Wi-Fi 输用户名密码 | 域账号认证 |
| 登录 OA/邮箱/Git | SSO 单点登录（基于域账号） |
| 密码过期要改密码 | 域控的密码策略 |
| 新电脑自动装了一堆软件 | 域控的组策略（GPO） |

### 你需要知道的

- **域账号** = 你在公司的唯一身份，通常是 `工号@company.com` 或 `COMPANY\工号`
- **密码策略**：通常要求定期修改、复杂度要求（大小写+数字+特殊字符）
- **组策略（GPO）**：IT 可以通过域控统一配置所有电脑的设置（壁纸、软件安装、USB 禁用等）
- **Mac/Linux 加域**：可以通过 LDAP 或第三方工具加入域，但不如 Windows 原生

## 11.8 上网行为管理

### 公司能看到什么？

| 你的行为 | 公司能看到吗？ |
|---|---|
| 访问了哪些网站（域名） | ✅ 能（DNS 日志/代理日志） |
| HTTPS 网页的具体内容 | ⚠️ 看情况（见下文） |
| 用了多少流量 | ✅ 能 |
| 什么时间上网 | ✅ 能 |
| 聊天内容（微信等） | ❌ 通常不能（端到端加密） |

### HTTPS 流量检查

有些公司会部署 **SSL 中间人解密**：
1. 在你电脑上安装公司的 CA 根证书
2. 防火墙/代理用这个证书"冒充"目标网站
3. 解密你的 HTTPS 流量，检查后再加密转发

🔑 **如果你的电脑是公司发的，并且安装了公司的证书，那 HTTPS 内容理论上公司是能看到的。**

**怎么判断：**
```
浏览器打开任意 HTTPS 网站 → 点击地址栏的锁 → 查看证书
如果证书颁发者是你公司的名字（而不是 Let's Encrypt/DigiCert 等）
→ 说明公司在做 SSL 解密
```

### 常见限制

| 限制 | 说明 |
|---|---|
| 网站黑名单 | 某些网站打不开（视频、游戏、社交） |
| 应用限制 | 不能装某些软件 |
| USB 禁用 | U 盘插上没反应 |
| 打印审计 | 打印内容被记录 |
| 截屏水印 | 截图自动带上你的工号 |

## 11.9 在公司网络中干活的实用技巧

### 技巧 1：搞清楚网络拓扑

```bash
# 查看你的 IP 和网关
ifconfig / ip addr
route -n / netstat -rn

# 查看 DNS 服务器
cat /etc/resolv.conf          # Linux
scutil --dns | head -20       # Mac

# 知道这些信息，排查问题时才有方向
```

### 技巧 2：内部系统访问不了

```
排查步骤：
1. ping 目标 IP → 不通？→ VLAN 隔离，找 IT 开权限
2. ping 通但端口不通 → 防火墙规则，找 IT
3. 端口通但服务报错 → 应用层问题，找对应系统管理员
4. 域名解析不了 → 公司内部 DNS 没有这条记录，找 IT
```

### 技巧 3：VPN 连接后的网络问题

```bash
# VPN 连上后某些网站打不开
# 可能是 DNS 被切换到公司内部 DNS
cat /etc/resolv.conf
# 如果 nameserver 变成了公司的 IP，外部域名可能解析不了

# 解决：手动加 DNS
# Mac：系统偏好设置 → 网络 → VPN 接口 → DNS → 添加 8.8.8.8

# VPN 连上后网速很慢
# 可能是全局模式，所有流量都走公司
# 解决：问 IT 能不能改成分流模式
```

### 技巧 4：代理环境下的开发

很多公司要求所有流量走 HTTP 代理：

```bash
# 查看公司代理地址（通常 IT 会告诉你，或者看系统设置）
# 假设是 proxy.company.com:8080

# 终端配置代理
export http_proxy=http://proxy.company.com:8080
export https_proxy=http://proxy.company.com:8080
export no_proxy=localhost,127.0.0.1,*.company.com   # 内网不走代理

# Git 配置代理
git config --global http.proxy http://proxy.company.com:8080

# Docker 配置代理
# 编辑 ~/.docker/config.json
{
  "proxies": {
    "default": {
      "httpProxy": "http://proxy.company.com:8080",
      "httpsProxy": "http://proxy.company.com:8080",
      "noProxy": "localhost,127.0.0.1,*.company.com"
    }
  }
}

# pip 配置代理
pip install --proxy http://proxy.company.com:8080 package_name

# npm 配置代理
npm config set proxy http://proxy.company.com:8080
```

### 技巧 5：申请权限的正确姿势

找 IT 开权限时，提供清晰的信息能大大加快处理速度：

```
❌ 错误示范："我访问不了 xxx 系统，帮我看看"

✅ 正确示范：
"你好，我需要从我的工位（IP: 192.168.10.50，VLAN 10）
 访问研发服务器 10.20.0.100 的 8080 端口（TCP）。
 目前 telnet 10.20.0.100 8080 显示 Connection timed out，
 怀疑是防火墙/VLAN 策略限制。
 请帮忙开通访问权限，谢谢。"
```

## 本章小结

| 概念 | 一句话 |
|---|---|
| VLAN | 在一个物理网络上划分多个虚拟网络，互相隔离 |
| 802.1X | 企业 Wi-Fi 认证，每人独立账号 |
| 企业 VPN | 远程接入公司内网的加密隧道 |
| 域控（AD） | 公司的中央账号管理，一个账号登录所有系统 |
| 上网行为管理 | 公司可以看到你访问了哪些网站 |
| 代理 | 公司流量可能强制走 HTTP 代理 |

**🔑 核心认知：** 公司网络的核心思路是"**控制与审计**"——控制谁能访问什么，记录谁访问了什么。理解这个思路，你就能理解公司网络的各种限制和配置。

## 📦 扩展区域

### NAC（网络准入控制）

有些公司要求设备通过"健康检查"才能接入网络：
- 操作系统是否最新？
- 杀毒软件是否安装？
- 是否安装了公司要求的安全软件？

不合规的设备会被放到"隔离 VLAN"，只能访问补丁服务器。

### RADIUS 服务器

802.1X 认证背后通常是 RADIUS 服务器，它负责验证用户身份并决定把用户分配到哪个 VLAN。

### 零信任网络（Zero Trust）

传统企业网络：内网 = 可信，外网 = 不可信。
零信任：**不信任任何人**，每次访问都要验证身份和权限。

代表产品：Google BeyondCorp、Cloudflare Access、Tailscale。

越来越多的公司在从传统 VPN 模式转向零信任模式。
