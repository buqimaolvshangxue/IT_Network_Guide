# 第 1 章 — 设备与标识：网络里的"谁是谁"

> 前置知识：第 0 章（全局地图）

## 开篇：这章要解决什么问题？

想象一下，你家里有手机、电脑、平板、智能音箱，它们都连着同一个 WiFi。路由器怎么知道该把哪个数据包发给谁？你的电脑怎么知道隔壁邻居家的电脑不是你家的设备？

**这章就是解决"网络里怎么认人"的问题。**

读完这章，你能做到：
- 看懂路由器后台那些 `192.168.1.100`、`255.255.255.0` 是什么意思
- 知道为什么有的 IP 地址是"内网"，有的是"公网"
- 理解设备是怎么找到"出口"（网关）的
- 能用命令查看自己设备的 MAC 地址、IP 地址、网关

## 🚦 新手入口（先抓主线）

### 30 秒先懂

- `MAC` 是设备网卡的硬件身份（局域网里认人）
- `IP` 是网络位置地址（网络里找路）
- `子网掩码` 用来判断"是不是同一局域网"
- `网关` 是去外网的默认出口

### 最小可用（先会判断）

1. 先看 `MAC vs IP` 的区别（别混）。
2. 再看 `子网掩码 + 网关`（决定走直连还是走出口）。
3. 最后跑一次命令，看自己机器的 `IP/MAC/网关`。

### 阅读顺序（建议）

- `MAC → IP → 子网掩码 → 网关 → IPv6`
- 本章优先建立判断能力，不要求死记概念名词

## 🔑 MAC 地址：设备的身份证号

### 一句话概括
**MAC 地址是网卡出厂时就烧录好的硬件地址，全球唯一，像身份证号一样。**

### 长什么样？
```
AA:BB:CC:DD:EE:FF
```
- 6 组数字，每组 2 位十六进制（0-9、A-F）
- 用冒号或短横线分隔，比如 `00:1A:2B:3C:4D:5E` 或 `00-1A-2B-3C-4D-5E`
- 前 3 组（AA:BB:CC）是厂商代码，后 3 组（DD:EE:FF）是设备序列号
  > 厂商代码由 IEEE（电气电子工程师学会）统一分配和维护。比如苹果的厂商代码是 `A4:83:E7` 开头，看到这个前缀就知道是苹果设备。

### 🔗 生活类比
就像你的身份证号：
- **出生就有**：网卡出厂时就写死了
- **全国唯一**：理论上全球没有重复的（实际上可以软件修改，但出厂时确实唯一）
- **不会变**：换城市、换工作，身份证号不变；设备换网络，MAC 地址也不变

### 日常什么时候碰到？
1. **路由器后台做 DHCP 绑定**（DHCP = Dynamic Host Configuration Protocol，自动分配 IP 地址的协议，详见第 2 章）：想让你的电脑每次都拿到固定 IP（比如 `192.168.1.100`），就需要填写电脑的 MAC 地址
2. **公司 WiFi 做设备白名单**：只允许特定 MAC 地址的设备连接
3. **查看连接设备列表**：路由器显示"有 5 台设备连接"，每台设备都有一个 MAC 地址

### 💻 动手验证：查看自己的 MAC 地址

**Mac 系统：**
```bash
ifconfig en0 | grep ether
```
输出示例：
```
ether a4:83:e7:12:34:56
```
> `en0` 通常是 WiFi 网卡，`en1` 可能是有线网卡

**Linux 系统：**
```bash
ip link show | grep link/ether
```
或者：
```bash
ifconfig | grep ether
```
输出示例：
```
link/ether 00:1a:2b:3c:4d:5e brd ff:ff:ff:ff:ff:ff
```

⚠️ **易错点**：
- 一台设备可能有多个 MAC 地址（WiFi 网卡一个、有线网卡一个、蓝牙一个）
- MAC 地址可以通过软件修改（叫"MAC 地址欺骗"），但出厂时确实是唯一的

## 🔑 IP 地址：网络里的门牌号

### 一句话概括
**IP 地址是设备在网络中的地址，像门牌号一样，用来找到设备的位置。**

### 🔗 生活类比：内网 IP vs 公网 IP
想象你住在一个小区：
- **内网 IP**：你家在小区里的门牌号，比如"3 栋 2 单元 501"
  - 只在小区内有效
  - 隔壁小区也可能有"3 栋 2 单元 501"，不冲突
  - 小区外的人用这个地址找不到你

- **公网 IP**：你们小区的街道地址，比如"人民路 123 号"
  - 全城唯一
  - 外面的快递员用这个地址能找到你们小区
  - 但找不到具体哪一户（需要小区保安/路由器转发）

### 内网 IP 地址段（重要！）
看到这些开头的 IP，就知道是内网地址：
- `192.168.x.x`（最常见，家用路由器默认）
- `10.x.x.x`（大公司内网常用）
- `172.16.x.x` 到 `172.31.x.x`（中型网络）

> 🤔 **为什么偏偏是这三个段？**
> 这不是随便选的。一个叫 IETF（互联网工程任务组）的国际组织在 1996 年发布了一份标准文档（RFC 1918），专门"预留"了这三段地址，规定它们**只能在内网使用，不会出现在公网上**。就像城市规划时预留了"内部道路"，不编入全国公路编号一样。全世界的路由器都遵守这个约定：看到这三个段的地址，就知道是内网的，不会往公网上转发。

例子：
```
192.168.1.100   ← 内网 IP
192.168.0.50    ← 内网 IP
10.0.5.20       ← 内网 IP
8.8.8.8         ← 公网 IP（Google 的 DNS）
114.114.114.114 ← 公网 IP（国内常用 DNS）
```

### 公网 IP
- 运营商分配给你家路由器的地址
- 全世界唯一（理论上）
- 你家所有设备共享这一个公网 IP 出去上网

> 🤔 **公网 IP 是谁分配的？整个链条是这样的：**
> 1. **IANA**（互联网号码分配机构）—— 全球 IP 地址的"总管家"，把大段 IP 分给各大洲
> 2. **各大洲的区域注册机构**（如亚太地区的 APNIC）—— 把 IP 分给各国的运营商
> 3. **运营商**（电信、联通、移动）—— 把 IP 分配给你家的路由器
>
> 所以你家的公网 IP 是运营商"借"给你用的，不是你的。运营商随时可以换一个给你（这就是为什么公网 IP 通常是"动态"的）。

> 🤔 **公网 IP 是什么时候分配的？**
>
> 是在**路由器拨号连接运营商时**分配的：
> ```
> 路由器开机/重启
>     ↓
> 通过 PPPoE 拨号（用你在运营商办的宽带账号密码）
>     ↓
> 运营商验证通过
>     ↓
> 分配一个公网 IP 给路由器
> ```
>
> **什么时候会重新分配？**
> | 情况 | 会重新分配公网 IP 吗？ |
> |-----|---------------------|
> | 路由器重启 | 会（重新拨号） |
> | 断网后恢复 | 会（重新拨号） |
> | 路由器设置里点"重新连接" | 会 |
> | 你的手机连上 WiFi | **不会**（手机只拿内网 IP） |
>
> **关键区分**：
> - 公网 IP → 给路由器的，拨号时分配
> - 内网 IP → 给你的设备的，连 WiFi 时由 DHCP 分配

### 💻 动手验证：查看 IP 地址

#### 1. 查看内网 IP

**Mac 系统：**
```bash
ifconfig en0 | grep "inet "
```
输出示例：
```
inet 192.168.1.105 netmask 0xffffff00 broadcast 192.168.1.255
```
> `192.168.1.105` 就是你的内网 IP

**Linux 系统：**
```bash
ip addr show | grep "inet "
```
或者：
```bash
hostname -I
```
输出示例：
```
192.168.1.105
```

#### 2. 查看公网 IP

**Mac 和 Linux 通用：**
```bash
curl ifconfig.me
```
或者：
```bash
curl ip.sb
```
输出示例：
```
123.45.67.89
```
> 这就是你家路由器的公网 IP，全世界都能访问到（如果路由器开了端口转发）

⚠️ **易错点**：
- 大部分家庭宽带的公网 IP 是**动态的**（每次重启路由器可能会变）
- 有些运营商给的是"假公网 IP"（运营商级 NAT），你拿到的其实还是内网 IP

### 特殊 IP：127.0.0.1（localhost）—— "我自己"

有一个特殊的 IP 地址你需要知道：**127.0.0.1**

**一句话概括**：在任何电脑上，`127.0.0.1` 都代表"这台电脑自己"。

```
在你电脑上访问 127.0.0.1 → 访问你自己的电脑
在服务器上访问 127.0.0.1 → 访问服务器自己
```

#### 🔗 生活类比

就像说"我"：
- 你说"我" → 指的是你自己
- 你朋友说"我" → 指的是他自己
- `127.0.0.1` 就是网络世界的"我"

#### 常见用途

| 场景 | 例子 |
|-----|------|
| 本地开发测试 | 浏览器访问 `http://127.0.0.1:3000` 看自己跑的网站 |
| 数据库连接 | 程序连 `127.0.0.1:3306` 访问本机的 MySQL |
| 代理配置 | `http_proxy=http://127.0.0.1:8080` 用本机的代理 |

#### localhost 和 127.0.0.1 是一回事

`localhost` 是 `127.0.0.1` 的域名别名，两个写法完全等价：
```
http://localhost:3000
http://127.0.0.1:3000
```

#### ⚠️ 易错点

**127.0.0.1 只能在本机访问，外面的人访问不了。**

```
你电脑上跑了个网站，监听 127.0.0.1:3000
    ↓
你同事想用他的电脑访问 你的IP:3000
    ↓
❌ 访问不了！因为 127.0.0.1 只接受本机的请求
```

想让别人访问？需要让程序监听 `0.0.0.0`（所有网卡）或具体的内网 IP（如 `192.168.1.100`）。

## 🔑 子网掩码与 CIDR（Classless Inter-Domain Routing，无类别域间路由）：怎么判断"是不是一家人"

### 一句话概括
**子网掩码用来划分"哪些 IP 是同一个局域网的"，就像邮编的前几位决定你在哪个区。**

### 🤔 为什么需要子网掩码？

你可能会想：既然 IP 地址已经能标识设备了，为什么还需要子网掩码？

**答案：IP 地址本身不包含"哪部分是网络号、哪部分是主机号"的信息。**

打个比方：电话号码 `010-12345678`，你知道 `010` 是区号、`12345678` 是本地号码，是因为你知道区号的规则。但如果把横线去掉变成 `01012345678`，没有规则你就分不清哪部分是区号了。

子网掩码就是这个"规则"——它告诉电脑：IP 地址的前多少位是"网络号"（相当于区号），剩下的是"主机号"（相当于本地号码）。

**没有子网掩码会怎样？** 电脑不知道目标 IP 是在同一个局域网还是在外面。如果在同一个局域网，直接用 ARP 找 MAC 地址发送；如果在外面，就要发给网关。这个判断完全依赖子网掩码。

### 最常见的：/24 和 255.255.255.0

**255.255.255.0** 就是 **/24**，它们是一个意思：
- 前 24 位是"网络号"（决定你在哪个局域网）
- 后 8 位是"主机号"（局域网内的设备编号）

> 🤔 **255.255.255.0 这个数字怎么来的？为什么是 255？**
> 
> 这需要知道一点二进制（很简单）：
> - 计算机用二进制（0 和 1）表示数字
> - **255** 用二进制写就是 **11111111**（8 个 1）
> - **0** 用二进制写就是 **00000000**（8 个 0）
> 
> 所以 `255.255.255.0` 用二进制写出来就是：
> ```
> 11111111.11111111.11111111.00000000
> ├── 前 24 个 1 = 网络号 ──┤├ 8个0 ┤
> ```
> 
> **规则：1 对应的位置是"网络号"，0 对应的位置是"主机号"。** 数一下有多少个 1？24 个。所以叫 **/24**。
> 
> 同理：
> - `255.255.0.0` = 16 个 1 + 16 个 0 = **/16**
> - `255.0.0.0` = 8 个 1 + 24 个 0 = **/8**

### 🔗 生活类比
邮编 `100000`：
- 前 3 位 `100` 代表北京市
- 后 3 位 `000` 代表具体区域

IP 地址 `192.168.1.100`（子网掩码 255.255.255.0）：
- 前 24 位 `192.168.1` 代表这个局域网
- 后 8 位 `100` 代表这台设备

### 怎么判断两个 IP 是否在同一个局域网？

**规则：网络号相同 = 同一个局域网**

例子（子网掩码都是 255.255.255.0）：
```
192.168.1.100  ← 网络号：192.168.1
192.168.1.200  ← 网络号：192.168.1  ✅ 同一个局域网

192.168.1.100  ← 网络号：192.168.1
192.168.2.100  ← 网络号：192.168.2  ❌ 不同局域网
```

### 日常场景
打开路由器后台，你会看到：
```
IP 地址：192.168.1.1
子网掩码：255.255.255.0
```
这意味着：
- 路由器的 IP 是 `192.168.1.1`
- 这个局域网可以容纳 `192.168.1.1` 到 `192.168.1.254` 共 254 台设备
- `192.168.1.0` 是网络地址（不能分配给设备）
- `192.168.1.255` 是广播地址（不能分配给设备）

> 🤔 **路由器的 IP（192.168.1.1）是怎么来的？**
> 
> 答案：**出厂默认设置的，可以改。**
> 
> - 路由器厂商在出厂时预设了一个内网 IP（通常是 `192.168.1.1` 或 `192.168.0.1`）
> - 这个 IP 同时也是局域网内所有设备的"网关"（出口）
> - 为什么通常是 `.1`？纯粹是约定俗成，方便记忆。你完全可以在路由器后台改成 `.254` 或其他数字
> - 不同品牌的默认值不同：TP-Link 常用 `192.168.1.1`，小米常用 `192.168.31.1`，华为常用 `192.168.3.1`
> 
> **而你的手机、电脑的内网 IP 是路由器通过 DHCP 协议自动分配的**（详见第 2 章）。路由器会从一个"地址池"（比如 192.168.1.100~200）里挑一个空闲的 IP 给你。

### 其他常见的子网掩码

| 子网掩码 | CIDR | 可用设备数 | 常见场景 |
|---------|------|-----------|---------|
| 255.255.255.0 | /24 | 254 台 | 家庭、小公司 |
| 255.255.0.0 | /16 | 65534 台 | 大公司 |
| 255.0.0.0 | /8 | 1677 万台 | 超大型网络 |

⚠️ **日常只会碰到 /24**，记住这一个就够了！但要知道还有 /16、/8 等。

## 🔑 网关：局域网的"出口"

### 一句话概括
**网关是局域网的出口，通常就是路由器的 IP 地址（如 192.168.1.1）。**

### 🤔 为什么需要网关？不能直接发给目标吗？

**答案：局域网内的设备只能通过 MAC 地址直接通信，而 MAC 地址只在局域网内有效。**

你的电脑要访问百度（IP: 110.242.68.66），但你的电脑和百度的服务器不在同一个局域网。你的电脑没办法用 ARP 广播去问"谁是 110.242.68.66"——因为百度的服务器根本不在你的局域网里，听不到你的广播。

所以你需要一个"中转站"——网关（路由器）。它一只脚踩在你的局域网里（内网 IP），另一只脚踩在外面的网络上（公网 IP），能帮你把数据"递"出去。

**简单说：局域网内靠 MAC 地址直接送，局域网外靠网关帮你转发。**

### 🔗 生活类比
小区大门口的保安亭：
- 你要寄快递到外地，必须先交给保安亭（网关）
- 保安亭知道怎么把快递送到外面的世界（公网）
- 外面的快递要进小区，也得先经过保安亭

### 工作原理
你的电脑要访问 `www.baidu.com`：
1. 电脑发现 `www.baidu.com` 的 IP（比如 `110.242.68.66`）不在同一个局域网
2. 电脑把数据包丢给网关（`192.168.1.1`）
3. 网关（路由器）把数据包转发到公网
4. 百度的服务器回复数据，先到网关，网关再转发给你的电脑

### 💻 动手验证：查看网关地址

**Mac 系统：**
```bash
netstat -nr | grep default
```
输出示例：
```
default            192.168.1.1        UGSc           en0
```
> `192.168.1.1` 就是网关

**Linux 系统：**
```bash
ip route | grep default
```
或者：
```bash
route -n | grep UG
```
输出示例：
```
default via 192.168.1.1 dev eth0
```

### 日常场景
- 路由器后台显示"默认网关"或"Gateway"
- 你的设备网络设置里有"路由器"或"网关"选项
- 网关通常是 `192.168.1.1` 或 `192.168.0.1`（路由器的 IP）

⚠️ **易错点**：
- 网关不一定是 `.1` 结尾，但大部分路由器默认是
- 如果网关配置错了，你能 ping 通局域网内的设备，但上不了网

### 🏠 网关 vs 路由器：家庭 vs 公司

很多人分不清网关和路由器，因为家用网络里它们是同一个设备。但概念上不一样：

| | 网关 | 路由器 |
|--|------|-------|
| 是什么 | 一个**角色**（网络的出口） | 一个**设备**（负责转发数据） |
| 类比 | "保安"这个职位 | "张三"这个人 |

**家庭网络：网关 = 路由器（同一个设备）**

```
┌─────────────────────────────┐
│       你家的路由器          │
│       (兼任网关角色)        │
│                             │
│  内网 IP: 192.168.1.1      │ ← 朝内（网关地址）
│  外网 IP: 123.45.67.89     │ ← 朝外（公网 IP）
│                             │
│  一台设备干了：网关、路由、NAT、DHCP、WiFi...
└─────────────────────────────┘
```

所以你的电脑配置里，网关就是路由器的 IP（`192.168.1.1`）。

**公司网络：网关和路由器可能分开**

```
你的电脑
    ↓
网关/防火墙（IP: 192.168.1.1）← 你的默认网关（只有内网 IP）
    ↓
核心路由器（IP: 10.0.0.1）    ← 在另一个网段
    ↓
边界路由器（内网: 10.0.0.254，公网: 123.45.67.89）← 这个才有公网 IP
    ↓
互联网
```

公司里：
- **网关**是你电脑的"第一道门"，通常有内网 IP
- **边界路由器**是公司网络的"外门"，有公网 IP
- 中间可能隔着好几层设备

**关键理解**：

1. **网关**是一个角色，意思是"你这个网段的出口"
2. **路由器**是设备，负责在不同网络之间转发数据
3. 家里一台设备同时扮演这两个角色，所以容易混淆
4. 公网 IP 给谁，取决于谁**直接连着运营商**

## 🔑 IPv6 基础：下一代 IP 地址

### 一句话概括
**IPv6 是新版本的 IP 地址，比 IPv4 长得多，用来解决 IPv4 地址不够用的问题。**

### 长什么样？
```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```
- 8 组数字，每组 4 位十六进制
- 用冒号分隔
- 可以简写：连续的 0 可以省略，比如 `2001:db8:85a3::8a2e:370:7334`

### 为什么需要 IPv6？
IPv4 地址只有 43 亿个（2^32），全球 80 亿人，每人还有好几台设备，早就不够用了。

IPv6 地址有 340 万亿亿亿亿个（2^128），地球上每粒沙子都能分配一个 IP。

### 和 IPv4 的关系
- **共存，不是替代**：目前大部分网站同时支持 IPv4 和 IPv6
- **你的设备可能同时有两个 IP**：一个 IPv4（`192.168.1.100`），一个 IPv6（`fe80::1a2b:3c4d:5e6f:7890`）

### 💻 动手验证：查看 IPv6 地址

**Mac 系统：**
```bash
ifconfig en0 | grep inet6
```
输出示例：
```
inet6 fe80::1a2b:3c4d:5e6f:7890%en0 prefixlen 64 secured scopeid 0x4
inet6 2001:db8:85a3::8a2e:370:7334 prefixlen 64 autoconf secured
```

**Linux 系统：**
```bash
ip -6 addr show
```

### 日常碰到的场景
- 有些网站同时支持 IPv4 和 IPv6（比如 Google、百度）
- 你的路由器后台可能显示"IPv6 地址"
- 某些校园网、企业网已经全面启用 IPv6

⚠️ **目前大部分日常配置还是 IPv4，IPv6 了解即可。**

## 📊 MAC 地址 vs IP 地址：一张表看懂区别

| 对比项 | MAC 地址 | IP 地址 |
|-------|---------|---------|
| **是什么** | 硬件地址（网卡的身份证） | 网络地址（设备的门牌号） |
| **谁分配** | 厂商出厂时写死 | 路由器动态分配（DHCP）或手动配置 |
| **会不会变** | 一般不变（除非软件修改） | 经常变（换网络就变） |
| **作用范围** | 局域网内（同一个路由器下） | 全球（公网 IP）或局域网（内网 IP） |
| **格式** | `AA:BB:CC:DD:EE:FF` | `192.168.1.100`（IPv4）<br>`2001:db8::1`（IPv6） |
| **生活类比** | 身份证号 | 门牌号 |
| **谁用它找设备** | 路由器/交换机（局域网内） | 路由器/互联网（跨网络） |

### 🔗 它们怎么配合工作？
1. 你的电脑要访问 `www.baidu.com`
2. 电脑先通过 DNS 查到百度的 IP 地址（`110.242.68.66`）
3. 电脑发现这个 IP 不在同一个局域网，把数据包发给网关（`192.168.1.1`）
4. **这时候用 MAC 地址**：电脑在局域网内找到网关的 MAC 地址，把数据包发给它
5. 网关收到数据包，看到目标 IP 是 `110.242.68.66`，转发到公网
6. 数据包在公网上传输，**全程用 IP 地址**找路
7. 百度服务器回复数据，原路返回

**简单说：局域网内用 MAC 地址找设备，跨网络用 IP 地址找设备。**

## 本章小结

### 核心概念
1. **MAC 地址**：设备的身份证号，出厂就有，全球唯一，局域网内用它找设备
2. **IP 地址**：设备的门牌号，分内网 IP 和公网 IP
   - 内网 IP：`192.168.x.x`、`10.x.x.x`、`172.16-31.x.x`
   - 公网 IP：全世界唯一，运营商分配
3. **子网掩码**：判断两个 IP 是否在同一个局域网
   - `255.255.255.0` = `/24`（日常最常见）
4. **网关**：局域网的出口，通常是路由器的 IP（`192.168.1.1`）
5. **IPv6**：下一代 IP 地址，和 IPv4 共存

### 记住这些命令

| 功能 | Mac | Linux |
|-----|-----|-------|
| 查看 MAC 地址 | `ifconfig en0 \| grep ether` | `ip link show \| grep link/ether` |
| 查看内网 IP | `ifconfig en0 \| grep "inet "` | `hostname -I` |
| 查看公网 IP | `curl ifconfig.me` | `curl ifconfig.me` |
| 查看网关 | `netstat -nr \| grep default` | `ip route \| grep default` |
| 查看 IPv6 | `ifconfig en0 \| grep inet6` | `ip -6 addr show` |

### 判断技巧
- 看到 `192.168.x.x` / `10.x.x.x` / `172.16-31.x.x` → 内网 IP
- 看到 `255.255.255.0` → 子网掩码 /24
- 看到 `AA:BB:CC:DD:EE:FF` 格式 → MAC 地址
- 看到 `2001:xxxx:xxxx:...` 格式 → IPv6 地址

## 📦 扩展区域

### 1. ARP 协议：IP 地址怎么找到 MAC 地址？
你的电脑知道网关的 IP 是 `192.168.1.1`，但怎么知道它的 MAC 地址？

答案：**ARP 协议**（Address Resolution Protocol）
- 电脑在局域网内广播："谁是 `192.168.1.1`？请告诉我你的 MAC 地址！"
- 路由器回复："我是 `192.168.1.1`，我的 MAC 地址是 `AA:BB:CC:DD:EE:FF`"
- 电脑把这个对应关系缓存起来（ARP 缓存表）

**查看 ARP 缓存表：**
```bash
# Mac 和 Linux 通用
arp -a
```
输出示例：
```
? (192.168.1.1) at aa:bb:cc:dd:ee:ff on en0 ifscope [ethernet]
? (192.168.1.100) at 11:22:33:44:55:66 on en0 ifscope [ethernet]
```

### 2. DHCP：IP 地址是怎么自动分配的？
你的手机连上 WiFi，自动就有了 IP 地址（`192.168.1.105`），这是怎么做到的？

答案：**DHCP 协议**（Dynamic Host Configuration Protocol）
1. 手机连上 WiFi，广播："有人能给我分配一个 IP 吗？"
2. 路由器回复："给你 `192.168.1.105`，子网掩码 `255.255.255.0`，网关 `192.168.1.1`，DNS `8.8.8.8`"
3. 手机："好的，谢谢！"

这就是为什么你不用手动配置 IP 地址。

### 3. 为什么有时候 MAC 地址会变？
- **虚拟机**：每次创建虚拟机，会生成一个随机 MAC 地址
- **隐私保护**：iOS 14+ 和 Android 10+ 支持"随机 MAC 地址"功能，每次连 WiFi 用不同的 MAC（防止被追踪）
- **MAC 地址欺骗**：软件可以修改 MAC 地址（比如绕过路由器的设备限制）

### 4. 公网 IP 不够用，怎么办？
答案：**NAT**（Network Address Translation，网络地址转换）
- 你家所有设备共享一个公网 IP
- 路由器记住"哪个内网 IP 发出的请求"，收到回复后转发给对应设备
- 这就是为什么你家 10 台设备都能上网，但只有 1 个公网 IP

### 5. 为什么 ping 不通某些网站？
可能的原因：
1. **对方服务器禁用了 ICMP**（ping 用的协议）
2. **防火墙拦截了 ping 请求**
3. **网络不通**（真的连不上）

试试用 `curl` 或浏览器访问，如果能访问，说明只是禁用了 ping。

### 6. 内网穿透：没有公网 IP 怎么让外网访问？
如果你想在公司访问家里的电脑，但家里没有公网 IP，怎么办？

答案：**内网穿透**（常用工具：frp、ngrok、ZeroTier）
- 原理：借用一台有公网 IP 的服务器做中转
- 你的电脑连接到中转服务器，外网访问中转服务器，中转服务器转发给你的电脑

### 7. 子网划分：为什么大公司要分很多子网？
- **安全隔离**：财务部和技术部的网络隔离，防止互相访问
- **性能优化**：减少广播风暴（局域网内的广播只在同一个子网传播）
- **方便管理**：不同部门用不同的 IP 段，一眼就能看出是哪个部门的设备
