# 第 0 章 · 全局地图 —— 你的数据包走了哪些路？

> **本章目标：** 在脑子里建立一张"网络全景图"，知道从你敲下回车到网页出现，数据到底经过了哪些节点。后面所有章节都是在这张图上"放大"某个局部。

---

## 🚦 第一次读先看这里

- 你上网的一条主链路是：`设备 → 路由器 → 运营商 → 目标服务器`
- 网络知识最容易乱，是因为概念没放在同一张图里
- 这章的任务就是先搭好这张图，不要求一次记住所有缩写

### 阅读顺序（不焦虑）

## 先说几个你会反复看到的缩写

本教程会频繁出现一些英文缩写，第一次见到可能会懵。这里先给你一张"最小速查表"，后面每章还会详细讲：

| 缩写 | 全称 | 一句话解释 |
|---|---|---|
| **IP** | Internet Protocol（互联网协议） | 设备在网络中的"门牌号"，如 `192.168.1.100` |
| **DNS** | Domain Name System（域名系统） | 把 `baidu.com` 翻译成 IP 地址的"电话簿" |
| **DHCP** | Dynamic Host Configuration Protocol（动态主机配置协议） | 自动给设备分配 IP 地址的"前台登记员" |
| **NAT** | Network Address Translation（网络地址转换） | 把一个 IP 地址换成另一个，最常见是把私网 IP 换成公网 IP |
| **HTTP** | HyperText Transfer Protocol（超文本传输协议） | 浏览器和网站之间"说话的语言"（不加密版） |
| **HTTPS** | HTTP + Secure | HTTP 的加密版，网址前面有 🔒 的就是 |
| **TCP** | Transmission Control Protocol（传输控制协议） | 保证数据完整送达的"挂号信" |
| **UDP** | User Datagram Protocol（用户数据报协议） | 追求速度不保证送达的"明信片" |
| **SSH** | Secure Shell（安全外壳协议） | 远程登录服务器的加密通道 |
| **VPN** | Virtual Private Network（虚拟专用网络） | 在公网上建一条加密隧道，像直接插网线到远程网络 |
| **TLS** | Transport Layer Security（传输层安全） | HTTPS 底层用的加密技术 |
| **ISP** | Internet Service Provider（互联网服务提供商） | 电信、联通、移动这些运营商 |
| **CDN** | Content Delivery Network（内容分发网络） | 在各地部署缓存服务器，让你访问最近的那个 |
| **MAC** | Media Access Control（媒体访问控制） | 网卡出厂时烧录的硬件地址，全球唯一，像设备的"身份证号" |
| **ARP** | Address Resolution Protocol（地址解析协议） | 在局域网内通过 IP 地址查找对应 MAC 地址的协议 |
| **CIDR** | Classless Inter-Domain Routing（无类别域间路由） | 用 `/24` 这样的写法表示子网掩码，比 `255.255.255.0` 更简洁 |
| **IANA** | Internet Assigned Numbers Authority（互联网号码分配机构） | 管理全球 IP 地址和端口号分配的国际组织 |
| **FTP** | File Transfer Protocol（文件传输协议） | 在网络上传输文件的协议（古老但仍在用） |
| **SFTP** | SSH File Transfer Protocol（SSH 文件传输协议） | 基于 SSH 加密的文件传输，FTP 的安全替代品 |

> 💡 不用现在就记住，遇到不认识的缩写随时翻回这里，或者查附录 C 术语表。

## 0.1 为什么需要一张全局地图？

学网络最怕的就是：**每个概念都听过，但串不起来。**

DNS、IP、路由器、防火墙、NAT……单独看都懂，一出问题就懵。原因很简单——你脑子里没有一条**完整的数据流动路径**。

这章就干一件事：**画出这条路径，让你知道数据从哪来、到哪去、中间经过了谁。**

## 0.2 🔑 完整网络路径

当你在 Mac 上打开浏览器，输入 `baidu.com` 按回车，数据走的路是这样的：

```
你的 Mac                    你家                         外面的世界
┌──────────┐  Wi-Fi    ┌──────────┐  网线    ┌──────────┐
│  浏览器   │ ───────→ │  路由器   │ ───────→ │   光猫    │
│ (应用层)  │          │(分配IP/  │          │(光信号↔  │
│          │          │ 防火墙)  │          │ 电信号)  │
└──────────┘          └──────────┘          └──────────┘
                                                  │
                                                  ↓
                        ┌──────────┐          ┌──────────┐
                        │  骨干网   │ ←─────── │  运营商   │
                        │(跨省/跨国│          │(电信/联通/│
                        │ 高速公路) │          │ 移动)    │
                        └──────────┘          └──────────┘
                              │
                              ↓
                        ┌──────────┐          ┌──────────┐
                        │  机房/CDN │ ───────→ │  服务器   │
                        │(离你最近 │          │(百度的   │
                        │ 的节点)  │          │ 电脑)    │
                        └──────────┘          └──────────┘
```

**一句话总结：** Mac → Wi-Fi → 路由器 → 光猫 → 运营商 → 骨干网 → 机房 → 服务器

## 0.3 逐个节点解释

### 🖥️ 你的 Mac（起点）

**类比：** 你本人，想给远方的朋友寄一封信。

浏览器生成一个"请求"（相当于写好信），交给操作系统的网络模块去发送。

> 📖 详见 → 第 1 章（设备与标识）

### 📶 Wi-Fi / 网线

**类比：** 从你家门口到小区信箱的那段路。

Wi-Fi 就是无线版的"网线"，把数据从 Mac 送到路由器。

> 📖 详见 → 第 6 章（网络设备）

### 🔀 路由器

**类比：** 小区门卫 + 邮局分拣员。

路由器干了很多事：
- **分配 IP**（DHCP）—— 给你的设备一个"门牌号"
- **转发数据**（路由）—— 决定数据往哪走
- **地址转换**（NAT）—— 把 IP 地址换成另一个（常见是私网换公网）
- **安全过滤**（防火墙）—— 挡住不该进来的东西

> 📖 详见 → 第 2 章（DHCP）、第 4 章（NAT/防火墙）、第 6 章（网络设备）

### 📡 光猫

**类比：** 翻译官，把"光语言"翻译成"电语言"。

光纤入户的那根线接到光猫，光猫把光信号转成电信号，路由器才能用。

> 📖 详见 → 第 6 章（网络设备）、第 7 章（家庭网络配置）

### 🏢 运营商（ISP）

**类比：** 邮政系统的本地分局。

中国电信、联通、移动——它们拥有从你家到互联网的"最后一公里"线路，也负责给你分配公网 IP。

### 🌐 骨干网

**类比：** 高速公路网。

连接各个城市、各个国家的超高速光纤网络。你访问国外网站时，数据就是走骨干网出去的。

### 🏭 机房 / CDN

**类比：** 各地的仓库/分店。

大网站会在全国各地部署 CDN（Content Delivery Network，内容分发网络），让你访问离你最近的服务器，速度更快。

### 💻 服务器（终点）

**类比：** 你朋友本人，收到信并回信。

百度的服务器收到你的请求，把网页内容打包发回来，沿着原路返回到你的浏览器。

## 0.4 极简网络分层

网络工程师把上面这条路分成了几"层"，每层管一件事：

```
┌─────────────────────────────────────────────┐
│  应用层    │ 你说什么？  │ HTTP、HTTPS、SSH、DNS │  → 第 3 章
├─────────────────────────────────────────────┤
│  传输层    │ 怎么送？    │ TCP（可靠）、UDP（快） │  → 第 3 章
├─────────────────────────────────────────────┤
│  网络层    │ 送到哪？    │ IP 地址、路由         │  → 第 1 章
├─────────────────────────────────────────────┤
│  链路/物理层│ 用什么送？  │ Wi-Fi、网线、光纤     │  → 第 6 章
└─────────────────────────────────────────────┘
```

🔑 **你不需要背 OSI（Open Systems Interconnection，开放系统互联）七层模型。** 日常 IT 工作只需要理解这四层就够了：
- **应用层**：浏览器、SSH 客户端等软件说的"语言"
- **传输层**：TCP 保证数据完整到达，UDP 追求速度
- **网络层**：IP 地址决定数据送到哪台机器
- **物理层**：实际的线缆和无线信号

## 0.5 🔑 完整例子：输入 baidu.com 按回车后发生了什么？

这是面试常考题，也是理解网络最好的例子。我们用大白话走一遍：

### 第 1 步：DNS 查询 —— "baidu.com 的 IP 是多少？"

浏览器不认识域名，只认识 IP 地址。所以第一件事是问 DNS 服务器：

```
你的 Mac → DNS 服务器：baidu.com 的 IP 是多少？
DNS 服务器 → 你的 Mac：是 39.156.66.10
```

> 📖 详见 → 第 2 章（DNS）

### 第 2 步：建立连接 —— "我要和 39.156.66.10 的 443 端口通话"

> 💡 **端口是什么？** 一台服务器上同时运行着很多服务（网页、邮件、数据库……），端口号就是区分这些服务的"房间号"。443 是 HTTPS 网页服务的标准房间号。详见第 3 章。

知道了 IP，浏览器通过 TCP 协议和百度服务器建立连接（三次握手）。因为是 HTTPS，还要进行 TLS 加密握手。

> 📖 详见 → 第 3 章（TCP）、第 5 章（HTTPS/TLS）

### 第 3 步：发送请求 —— "给我首页"

```
GET / HTTP/1.1
Host: www.baidu.com
```

浏览器发出 HTTP 请求，说"我要你的首页"。

### 第 4 步：数据包的旅程

这个请求被打包成一个个"数据包"，每个包上写着：
- **源 IP**：你的公网 IP（路由器 NAT 转换后的）
- **目标 IP**：39.156.66.10
- **源端口**：随机的（比如 52341）—— 操作系统自动分配的临时端口，用完就释放，就像打电话时你的电话号码
- **目标端口**：443（HTTPS）—— 百度服务器上"网页服务"的房间号

数据包经过：Mac → 路由器（NAT）→ 光猫 → 运营商 → 骨干网 → 百度机房

> 📖 详见 → 第 1 章（IP）、第 4 章（NAT）

### 第 5 步：服务器响应 —— "给你网页内容"

百度服务器处理请求，把网页内容打包发回来（HTML 是网页结构、CSS 是样式、JS 是交互逻辑），沿原路返回。

### 第 6 步：浏览器渲染

你的 Mac 收到响应，浏览器把 HTML 渲染成你看到的百度首页。

**整个过程不到 1 秒。**

## 0.6 💻 动手验证：traceroute

想亲眼看看数据经过了哪些节点？用 `traceroute` 命令：

```bash
# Mac
traceroute baidu.com

# Linux
traceroute baidu.com
# 或者用更好用的 mtr
mtr baidu.com
```

**输出示例：**
```
traceroute to baidu.com (39.156.66.10), 30 hops max
 1  192.168.1.1 (192.168.1.1)      1.234 ms   ← 你的路由器
 2  10.0.0.1 (10.0.0.1)            5.678 ms   ← 光猫/运营商
 3  61.152.x.x (61.152.x.x)       10.123 ms   ← 运营商节点
 4  202.97.x.x (202.97.x.x)       15.456 ms   ← 骨干网
 5  ...
 8  39.156.66.10 (39.156.66.10)    25.789 ms   ← 百度服务器
```

🔑 **每一行就是一"跳"（hop）**，对应数据经过的一个路由器/节点。数字越大，离你越远。

> 📖 详见 → 第 10 章（网络排查）会详细讲 traceroute 的用法

## 📦 扩展区域

> 以下内容在日常 IT 工作中不常用，但如果你感兴趣可以了解：

### OSI 七层模型（面试可能会问）

完整的 OSI 模型有 7 层，我们简化成了 4 层：

| OSI 层 | 名称 | 对应我们的简化层 |
|---|---|---|
| 7 | 应用层 | 应用层 |
| 6 | 表示层 | 应用层 |
| 5 | 会话层 | 应用层 |
| 4 | 传输层 | 传输层 |
| 3 | 网络层 | 网络层 |
| 2 | 数据链路层 | 物理层 |
| 1 | 物理层 | 物理层 |

> 日常工作中没人说"第 6 层有问题"，都是说"DNS 挂了"、"端口不通"这种具体描述。所以不用死记七层。

### 数据封装过程

数据从应用层往下走，每经过一层都会被"套一层信封"：

```
应用层数据 → [TCP头 + 数据] → [IP头 + TCP头 + 数据] → [以太网头 + IP头 + TCP头 + 数据]
```

这就像寄信：信纸（数据）→ 装进信封写上地址（IP）→ 装进邮袋标上路线（以太网帧）。
